# 家計分析ダッシュボード アプリケーション設計

## 概要

マネーフォワード Me の CSV エクスポートデータをもとに、家計支出を予算と比較・分析するダッシュボードを、Next.js 16 アプリケーションとして構築する。既存の HTML ダッシュボード（単一ファイル、状態非永続）をベースに、データベースによる永続化とインタラクティブな UI を実現する。

## 背景と課題

既存の HTML ダッシュボードには以下の課題がある。

- ページを更新すると予算マッピングの設定がリセットされる
- 予算の追加・編集がセッション内でしか保持されない
- CSV データの再取り込みのたびにビルドスクリプトを手動実行する必要がある

これらを解決するため、データベースで状態を永続化し、ブラウザから CSV インポートや予算管理ができるアプリケーションに移行する。

## 技術スタック

marumie プロジェクト（/Users/kenta/programing/marumie）と同じ技術スタックを採用する。

- Next.js 16（App Router）
- React 19
- TypeScript 5（strict モード）
- Tailwind CSS v4（`@import "tailwindcss"` 方式）
- Prisma ORM + PostgreSQL
- Recharts（チャート描画）
- pnpm（パッケージマネージャー）
- Biome（リンター / フォーマッター）

## アーキテクチャ方針

marumie の webapp 側のアーキテクチャに準拠する。

- 動的に更新する必要がある画面以外は、データ取得をサーバーコンポーネントに寄せる
- `"use client"` はクライアント側で動作する必然性（状態管理・ブラウザ API 利用等）がある場合のみ使用
- サーバーコンポーネントからのデータ取得は loaders に切り出して責務を分離
- サーバー側で動作する処理には `import "server-only"` を記述
- サーバーアクション（`"use server"`）は副作用を伴う操作のためだけに使い、`revalidatePath` による再検証を 1 セットで行う
- TypeScript の import は `@/` から始まる絶対パスを使用

## データベース設計

### ER 図（概念）

```
transactions ─── N:1 ─── budget_items ─── 1:N ─── budget_category_mappings
```

### transactions テーブル

マネーフォワード Me から取り込んだ個別取引を格納する。

| カラム | 型 | 説明 |
|---|---|---|
| id | String (UUID) | 主キー |
| date | DateTime (Date) | 取引日 |
| description | String | 内容（マネーフォワードの「内容」列） |
| amount | Int | 金額（円、絶対値で格納） |
| major_category | String | 大項目 |
| minor_category | String | 中項目 |
| institution | String? | 保有金融機関 |
| memo | String? | メモ |
| moneyforward_id | String? | マネーフォワード上の取引 ID |
| import_hash | String @unique | 重複排除用ハッシュ |
| created_at | DateTime | 作成日時 |
| updated_at | DateTime | 更新日時 |

インデックス: `date DESC`, `(major_category, minor_category)`

### budget_items テーブル

ユーザーが定義する予算項目を格納する。

| カラム | 型 | 説明 |
|---|---|---|
| id | String (UUID) | 主キー |
| name | String | 費目名（例: 電気代） |
| monthly_amount | Int | 月額予算（円） |
| cycle_type | CycleType (Enum) | 周期タイプ |
| sort_order | Int | 表示順 |
| created_at | DateTime | 作成日時 |
| updated_at | DateTime | 更新日時 |

CycleType enum: `monthly_fixed`, `monthly_variable`, `irregular_fixed`, `irregular_variable`

### budget_category_mappings テーブル

予算項目とマネーフォワードの中項目の紐づけを管理する。

| カラム | 型 | 説明 |
|---|---|---|
| id | String (UUID) | 主キー |
| budget_item_id | String (FK) | 予算項目への参照 |
| major_category | String | 大項目 |
| minor_category | String | 中項目 |
| created_at | DateTime | 作成日時 |

ユニーク制約: `(budget_item_id, major_category, minor_category)`

## データフロー

### CSV インポート

```
CsvUploadForm（client）
  → importCsv サーバーアクション
    → csv-parser.ts でパース（cp932 → UTF-8、フィルタリング）
    → ImportTransactionsUsecase
      → TransactionRepository.upsertMany()
    → revalidatePath("/dashboard")
```

フィルタ条件: `計算対象 = 1 AND 振替 = 0 AND 金額 < 0`（支出のみ）。金額は絶対値に変換して格納。

### 予算マッピング更新

```
MappingPanel（client）
  → updateMappings サーバーアクション
    → UpdateMappingUsecase
      → MappingRepository.replaceAll(budgetItemId, categories)
    → revalidatePath("/dashboard")
```

### ダッシュボードデータ取得

```
dashboard/page.tsx（server）
  → loadDashboardData ローダー
    → GetDashboardSummaryUsecase
      → TransactionRepository.getMonthlyAggregation()
      → TransactionRepository.getCategoryBreakdown()
      → BudgetRepository.findAllWithMappings()
      → 集計・KPI 計算
  → props としてクライアントコンポーネントに渡す
```

## CSV パーサー仕様

### 入力

- エンコーディング: cp932（Shift-JIS）
- カラム: 計算対象, 日付, 内容, 金額（円）, 保有金融機関, 大項目, 中項目, メモ, 振替, ID

### フィルタリング

- `計算対象 == 1`（計算対象の取引のみ）
- `振替 == 0`（振替を除外）
- `金額 < 0`（支出のみ）

### 変換

- 金額: 負の値を絶対値に変換して格納
- 日付: 文字列 → DateTime
- import_hash: `日付 + 内容 + 金額 + 金融機関 + ID` から SHA-256 で生成（重複排除）

### 収入データ

- `計算対象 == 1 AND 振替 == 0 AND 金額 > 0` のデータも取り込む（サマリー表示用）
- transactions テーブルに `is_income` フラグまたは金額の正負で区別

## 注意事項

- 家賃（70,700 円/月）と共同生活費（55,000 円/月）はマネーフォワードでは振替として処理されるため、CSV データには含まれない。予算項目としては定義するが、実績は 0 になる想定。
- チャートライブラリは Chart.js から Recharts に変更する（marumie との一貫性）。
- 単一アプリ構成とし、marumie のようなモノレポ構成にはしない。ただしコード品質パターン（リポジトリパターン、ローダー、ユースケース分離）は踏襲する。
